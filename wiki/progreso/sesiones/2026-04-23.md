## Lo que se Construyó

### Scaffold del proyecto
- `package.json` con todas las dependencias: Next.js 14.2.5, Supabase, Anthropic, Zustand, React Hook Form, Zod, TanStack Query, Recharts
- `tsconfig.json` en modo `strict: true` con path alias `@/*`
- `tailwind.config.js` con paleta de marca (indigo) y colores de semáforo
- `next.config.js` con serverActions y dominios de Supabase para imágenes

### Tipos del dominio
- `types/database.types.ts` — tipos generados del esquema Supabase: 5 tablas, 2 vistas, 7 enums
- `types/index.ts` — tipos de dominio: `Indicador`, `ElementoGestion`, `CriterioSemaforo`, `MetodoCalculo`, `Captura`, `Evidencia`, `Revision`, `AnalisisIA`, vistas

### Autenticación y Middleware
- `lib/supabase/client.ts` — cliente browser
- `lib/supabase/server.ts` — cliente server con manejo de cookies (try-catch para Server Components)
- `lib/supabase/middleware.ts` — lógica de protección: no-auth → /login, auth en /login → /dashboard
- `middleware.ts` (raíz) — wrapper que llama al middleware de Supabase

### App Router
- `app/layout.tsx` — layout raíz con fuentes Inter + Lexend
- `app/globals.css` — variables CSS, componentes globales Tailwind (.card, .badge-*, .btn-*, .input, .nav-item)
- `app/(auth)/login/page.tsx` + `login-form.tsx` — página de login con branding GDM
- `app/(auth)/logout/route.ts` — handler GET para cerrar sesión
- `app/(dashboard)/layout.tsx` — layout del dashboard: fetcha usuario con rol, renderiza Sidebar + Header
- `app/(dashboard)/dashboard/page.tsx` — redirect inteligente por rol a la ruta correspondiente
- Directorios placeholder: `/municipio`, `/indicadores`, `/evidencias`, `/revisiones`, `/estado`, `/admin`
- Directorios placeholder de API: `/api/capturas`, `/api/evidencias`, `/api/indicadores`

### Utilidades
- `lib/utils.ts` — `cn()`, `SEMAFORO_CONFIG`, `ESTADO_CAPTURA_CONFIG`, `ROL_LABEL`, `calcularSemaforoGestion()`, `formatFileSize()`, `formatFecha()`, `calcularPctAvance()`

### Componentes de Layout
- `components/layout/sidebar.tsx` — sidebar con navegación por rol y footer de perfil de usuario
- `components/layout/header.tsx` — header del dashboard

## Problemas Encontrados y Resueltos

1. **Tipos de cookies en Server Components**: El manejo de cookies en `lib/supabase/server.ts` requiere `try-catch` porque `cookies().set()` lanza en algunos contextos de Server Components — resuelto con manejo explícito.

2. **Tipo de rol en redirect**: El tipo `rol_usuario` del enum de Supabase necesita cast explícito al comparar con las constantes string — resuelto con cast `as RolUsuario`.

3. **Estructura del repo**: Los archivos inicialmente se crearon en un subdirectorio — movidos a la raíz con `2dc418d`.
